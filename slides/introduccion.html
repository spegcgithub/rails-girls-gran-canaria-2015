Virtualización y Linux

Virtualización

¿Qué es?

Creación de una versión «virtual» de un recurso tecnológico. Se puede virtualizar desde un disco duro hasta un servidor completo.

Simplificando (mucho)

Ejecución de un sistema operativo dentro de otro.

Simplificando (muchísimo)

¿Qué es?

Beneficios

  • Para nosotros:
    • Facilita las pruebas
    • Mantiene limpio el sistema base
    • Entornos casi idénticos a los de producción (en mi máquina funciona (TM))
  • En general:
    • Aprovechamiento de recursos (reducción de costes)
    • Mayor seguridad
    • Alta disponibilidad

Complejo

  • Es un asunto bastante complejo
  • Múltiples técnicas: virtualización completa, paravirtualización, asistida por hardware, ligera, etc.
  • El hardware ofrece extensiones para facilitarla (VT-x, AMD-V, VT-c, etc.)
  • Aún se sigue investigando
  • VMWare, VirtualBox, Xen, QEMU, KVM, LXC, Microsoft Hyper-V, etc.

Oracle VirtualBox

  • Una de las soluciones más populares
  • Interfaz cómoda y agradable
  • Multiplataforma
  • Software libre
  • Virtualización completa (+paravirtualización)

Linux

Sistemas operativos y aplicaciones

  • Un sistema operativo: conjunto de programas de un sistema informático que:
    • Gestiona los recursos físicos (hardware)
    • Ofrecer servicios a las aplicaciones
  • Ordenadores, teléfonos, enrutadores... todos tienen uno
  • El navegador, la suite de ofimática o el «Whatsapp» son aplicaciones y no se consideran parte del sistema operativo.

Linux es un sistema operativo

  • Sistema operativo
  • Nacido en 1991 de la mano de Linus Torvalds
  • Multiplataforma
  • Es software libre

No sólo en servidores

  • Servidores (Google o Facebook son buenos ejemplos)
  • Sistemas de escritorio (pequeño porcentaje del mercado)
  • Sistemas empotrados
  • Supercomputadores (97% en el Top 500 en noviembre de 2014)
  • ¿Alguien tiene un teléfono Android?

Ecosistema: núcleo y aplicaciones

  • Linus Torvalds coordina el desarrollo del núcleo
    • Cientos de desarrolladores (voluntarios y empresas)
    • Linus delega en gente de confianza
  • Existen entidades que desarrollan aplicaciones
    • Desarrolladores independientes, fundaciones, empresas
    • Desde pequeños proyectos...
    • ...hasta software muy complejo (Mozilla Firefox, LibreOffice.org, Ruby on Rails, etc.)

Ecosistema: distribuciones

  • Existen otras entidades que construyen productos combinando lo anterior
  • Simplificando, distribución = núcleo + aplicaciones + herramientas
  • Mucha variedad tanto desde el punto de vista técnico como filosófico
  • Algunos ejemplos: Ubuntu, openSUSE, Red Hat Linux, Debian GNU/Linux, etc.
